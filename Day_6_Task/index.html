Box Model Exercises with Tailwind CSS

Exploring borders, padding, margins, and box-sizing

Task 1: Different Border Widths and Colors

This box has unique border widths and colors on each side.

Using Tailwind classes: border-t-4 (top: 4px), border-b-8 (bottom: 8px), border-l-2 (left: 2px), border-r-4 (right: 4px), with colors like border-t-red-500.

Task 2: Adding Padding

No Padding
With Padding (p-6)

The second box uses p-6 (padding: 1.5rem or 24px on all sides), increasing the space inside the box. Compare it to the first box with no padding—content is pushed away from the edges, affecting the box’s internal layout.

Task 3: Adjusting Margins

Box 1 - No Margin
Box 2 - Margin Top (mt-8)
Box 3 - Margin Top (mt-4)

Margins control spacing outside the box. mt-8 (margin-top: 2rem or 32px) and mt-4 (margin-top: 1rem or 16px) space these boxes vertically, preventing them from touching.

Task 4: Box-Sizing: Border-Box

Default (content-box): 64px + padding + border
Border-Box: 64px total width

Both boxes use w-64 (width: 16rem or 256px). The first uses the default content-box, where padding (16px per side) and border (4px per side) add to the width, making it 288px total. The second uses box-border (box-sizing: border-box), keeping the total width at 256px, with padding and border included in the calculation.
